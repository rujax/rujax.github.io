WARNING

问题

最近给博客添加了一大波文章封面，测试时发现一个问题： 图片都是 PNG 格式的，每一个文件大小都超过了 2M。 我的博客对于图片的精细度并没有什么要求，只要能大致看清图片里是什么就行。 因此，这个大小的图片加载速度确实比较影响阅读体验。

考虑到预加载器的图片也因为网速的问题，时不时加载不出来， 于是下定决心将所有 PNG 格式的图片都转变为 WebP 格式。

解决

在 Windows 系统上批量提取 PNG 格式图片

由于此前使用命令行的生涯基本都是在 Linux 和 Mac 上，对于 Windows 系统上如何批量提取 PNG 格式图片不甚了解。 还好有万能的互联网，经过搜索查询到了 Windows 命令行上使用的命令 for 。

通过文档的示例可以得到基础的使用方式：

# *.jpg 的原因是因为发现博客里的图片除了 PNG 格式还有少许 JPG 格式的，一起处理了
$ for %f in (*.png *.jpg) do <command> [<commandlineoptions>]

简单解释下：for 循环遍历当前文件夹内所有扩展名为 .png 或者 .jpg 的文件，并将每次遍历的完整文件名存入变量 %f 中。

接下来，就是将格式转换的命令填入上述的 <command> [<commandlineoptions>] 中。

使用 FFmpeg

FFmpeg 是非常优秀的视频处理工具，同时它也拥有图片处理功能。

检测编码器

首先检查当前使用的 FFmpeg 是否含有 WebP 编码器：

$ ffmpeg.exe -loglevel quiet -encoders | grep webp
 V....D libwebp_anim         libwebp WebP image (codec webp)
 V....D libwebp              libwebp WebP image (codec webp)

格式转换

格式转换的命令就比较简单了：

# 新创建一个文件夹 output 用来存放转换后的图片
$ ffmpeg.exe -i input.png -c:v libwebp output/output.webp
  • -i：指定输入图片
  • -c:v：指定图像处理编码器
  • output/output.webp：指定输出图片

结合 for 命令 

$ for %f in (*.png *.jpg) do ffmpeg.exe -i %f -c:v libwebp output/<file name only>.webp

出现了新的问题，之前存储文件名的变量 %f 包含了文件名和文件扩展名，但是在这条命令中，我们只需要文件名。

查阅之前 for 命令的文档，找到使用 %~n<variable> 的方式可以只获得文件名，调整命令：

$ for %f in (*.png *.jpg) do ffmpeg.exe -i %f -c:v libwebp output/%~nf.webp

大功告成，顺利转换图片的格式。

使用 cwebp

除了使用 FFmpeg，Google 还提供了针对 WebP 一系列的工具。

结合 for 命令格式转换 

$ for %f in (*.png *.jpg) do cwebp.exe %f -o output/%~nf.webp

2个工具转换后的图片大小对比

首先要说的是，本文上述的转换方式都不是无损压缩（我需要的正是有损压缩

cwebp 工具提供了参数 -q 指定压缩系数，FFmpeg 转换格式后的 .webp 图片大小约等于使用 cwebp 指定参数 -q 75 的结果。

具体使用哪个工具，根据自己的需求就好。

Note

如果需要无损压缩的话，请使用 cwebp 工具并指定参数 -lossless。

结语

全站的图片都已经转换为 WebP 格式，肉眼可见的加载速度提升 <(￣︶￣)>
