WARNING

问题

我的博客是基于 Hugo 框架和 Reimu 主题搭建的。
其中 Hugo 使用的是 v0.159.2 版本，每次启动服务时，都能看到提示方法或字段已废弃的警告：

Warning

WARN deprecated: .Site.Languages was deprecated in Hugo v0.156.0 and will be removed in a future release. See https://discourse.gohugo.io/t/56732.
WARN deprecated: .Site.Data was deprecated in Hugo v0.156.0 and will be removed in a future release. Use hugo.Data instead.

本人是看到警告都无法忍受的那类人（笑
其实是没有拖延症，一想到万一哪天更新版本后出现异常报错就难以忍受（
因此，在博客基本搭建成形后，开始着手解决这2个警告。

.Site.Data

修复

通过上面的已废弃警告和 .Site.Data 的文档，可以轻松得知：直接使用 hugo.Data 文本匹配替换 即可。

.Site.Languages

.Site.Languages 的警告修复则要麻烦许多。

资料查询过程

已废弃警告的提示信息和 .Site.Languages 的文档，都指向了参考链接 https://discourse.gohugo.io/t/56732。

打开链接后，可以看到 Deprecated methods 标题下提供了一张在 Hugo v0.156.0 版本会被开始标记为废弃的方法清单：

点击 Example B 后，可以看到一段示例代码：

{{ with .Rotate "language" }}
  <nav class="language-switcher">
    <ul>
      {{ range . }}
        {{ if eq .Site.Language $.Site.Language }}
          <li class="active"><a aria-current="page" href="{{ .Permalink }}">{{ .Site.Language.LanguageName }}</a></li>
        {{ else }}
          <li><a href="{{ .Permalink }}">{{ .Site.Language.LanguageName }}</a></li>
        {{ end }}
      {{ end }}
    </ul>
  </nav>
{{ end }}

示例代码使用了 .Rotate 方法，展示了一个简单的语言切换器。
しかし、残念です。
这段示例代码并未展示如何获得原本 .Site.Languages 方法返回的 langs.Languages。

修复方案

查询多个 Hugo 的文档后，得到了一个修复方案。

  1. 调用函数 hugo.Sites 得到返回 page.Sites；

  1. 遍历 page.Sites 后提取每一个 page.siteWrapper 调用 Language 方法得到的 Language 对象。

代码应用

通过关键字定位后，可以发现2处使用 .Site.Languages 的地方。

// themes/reimu/layouts/partials/header.html
{{- $i18nMap := dict -}}
{{- range .Site.Languages -}}
  {{- $i18nMap = merge $i18nMap (dict .Lang .LanguageName) -}}
{{- end -}}

// themes/reimu/layouts/partials/head/config.html
{{- $i18n_languages := .Site.Languages -}}
{{- if hugo.IsMultilingual -}}
  {{- $siteConfig = merge $siteConfig (dict "i18n_languages" $i18n_languages) -}}
{{- end -}}

首先解决第1处。
从代码可以得知，构造了一个 map（变量名：$i18nMap），然后遍历配置的语言，将每个语言以 { .Lang: .LanguageName } 这样的键值对存入 $i18nMap。

执行代码，输出 $i18nMap 的值：

map["de":"Deutsch" "zh-cn":"简体中文"] // de 是用来测试代码新添加的配置

从 Site.Language 的文档中可得知 Lang 和 LanguageName 也已在 Hugo v0.158.0 之后标记为已废弃状态，

所以，顺手将这2个方法的调用也修改掉：

// themes/reimu/layouts/partials/header.html

{{- $i18nMap := dict -}}

// 修改之前
{{- range .Site.Languages -}}
  {{- $i18nMap = merge $i18nMap (dict .Lang .LanguageName) -}}
{{- end -}}

// 修改之后
{{- range hugo.Sites -}}
  {{- $i18nMap = merge $i18nMap (dict .Language.Name .Language.Label) -}}
{{- end -}}

输出 $i18nMap 的值与之前一致。

接着解决第2处。
从代码可以得知，最终会将 map["i18n_languages"]{.Site.Languages} 合并到 map $siteConfig 中。
接下来，$siteConfig 会在 JavaScript 中使用：

<script>
  window.siteConfig = JSON.parse({{ $siteConfig | jsonify }});
</script>

输出 siteConfig.i18n_languages 的值，可以得到 Array：

[
    {
        "Lang": "zh-cn",
        "Title": "",
        "Weight": 1,
        "Disabled": false,
        "Direction": "",
        "Label": "简体中文",
        "Locale": "zh-CN"
    },
    {
        "Lang": "de",
        "Title": "",
        "Weight": 2,
        "Disabled": false,
        "Direction": "ltr",
        "Label": "Deutsch",
        "Locale": "de-DE"
    }
]

修改代码

// themes/reimu/layouts/partials/head/config.html

// 修改之前
{{- $i18n_languages := .Site.Languages -}}

// 修改之后
{{- $i18n_languages := slice -}}
{{- range hugo.Sites -}}
  {{- $i18n_languages = $i18n_languages | append .Language -}}
{{- end -}}

输出修改后的代码，$i18n_languages 的值一致。

再次全局搜索 i18n_languages 可以看到在 themes/reimu/assets/js/global.d.ts 和 themes/reimu/assets/js/pjax.ts 也有使用。
其中，global.d.ts 中使用的还是旧字段 LanguageName 将其调整为 Label。

至此，2处调用都修改完毕。
