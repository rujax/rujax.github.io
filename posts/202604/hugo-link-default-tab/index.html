问题

Hugo 框架下，点击文章中的链接默认行为是：页面跳转。
我个人是不太喜欢这种默认行为，因为极大可能文章还没有读完，只是看到了想了解的内容链接，于是点击进入了新的页面。
在新的页面又有可能点击更多的链接，进入更多的新的页面，最后想要回到博客文章，要点无数次的后退，
甚至有可能新的页面点击链接的默认行为是 打开新标签页，
然后不小心关掉了最初跳转的页面，导致后退也退不回博客的情况出现 (⊙﹏⊙)

虽然我知道使用 鼠标中键点击 和 按住 CTRL 键鼠标左键点击 都可以在新标签页打开链接，
但这对于我而言，这并非习惯的行为，如果能修改点击链接的默认行为那就更好了。

解决过程

通过查询 Hugo 的 文档 ，找到了它在渲染 Markdown 文章中链接的钩子。

文档里提到了链接的各部分组件都传递给了渲染钩子模版，通过调用上下文中的属性可以获取到链接的各部分内容。

同时，在路径 layouts/_markup/render-link.html 文件里可以创建渲染钩子覆盖掉原始的渲染方式。

解决方案

创建文件 layouts/_markup/render-link.html，写入如下内容：

<a
    href="{{ .Destination | safeURL }}"
    {{ with .Title }} title="{{ . }}"{{ end }}
    {{ if strings.HasPrefix .Destination "http" }} target="_blank" rel="noopener noreferrer"{{ end }}
>
    {{ .Text | safeHTML }}
</a>

简单描述一下代码：

  1. 通过上下文属性 .Destination 获取到超链接目标并通过 safeURL 函数安全化显示；
  2. 使用 with 函数绑定上下文属性 .Title ，并且添加对 .Title 的存在判断，如果 .Title 存在，添加超链接标签的属性 title，值为 .Title；
  3. 调用函数 strings.HasPrefix 判断链接目标如果以 http 开头的话，添加超链接的属性 target 和 rel ：
    • target：设置打开超链接的位置，_blank 则表示新标签页打开
    • rel：设置当前文档与目标的关系
      • noopener 设置新打开的页面无法通过 window.opener 操作前页面
      • noreferrer 则不会泄漏前页面的 Referrer 的信息
  4. 使用上下文属性 .Text 获取到超链接的文本，并且调用函数 safeHTML 做安全化显示。

结语

大功告成！
