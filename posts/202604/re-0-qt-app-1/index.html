WARNING

写在安装之前

初识 Qt

大学时期非常喜欢《三国杀》端游的我在 Github 徜徉时，无意间邂逅了一个使用 Qt 实现，名为《太阳神三国杀》的开源项目，顿时惊为天人。
还依稀记得当时一遍遍地编译源码，不懈努力之下，终于解决了所有报错，在 Ubuntu 系统（PS：Windows 系统上可能容易很多）上编译成功时的喜悦之情。
那个时候，对 Qt 的印象就是：能写游戏（大概吧<(￣ˇ￣)/）。

再遇 Qt

再次遇到 Qt 已经是多年之后的事了。

我的主业是服务器后端开发，在工作的日常中也不可避免地接触到了 Html、CSS、JavaScript。
慢慢地，我成为了一名 Web 全栈开发工程师。
直到有一天，那是一个夕阳微斜映的黄昏，我坐在日复一日几无挑战默默耕耘的座位上，突然想起了自己当初是为什么开始学习编程。
小学时，第一次接触 Windows 95 和 98 时，眼前看到的一切无一不让我啧啧称奇。
没错，是客户端应用，是桌面上的各个客户端应用让我萌生了想要学习编程的念头。

于是，我开始了业余时间的客户端编程之旅（PS：开始做服务端开发也算是大学实习时误入歧途了╮(╯▽╰)╭）。
精挑细选之下，在数个能够实现客户端应用的框架中，与 Qt 再次相遇了。

终定 Qt

Qt 框架基于 C++ 开发。
众所周知，C++ 是一门极其复杂难以精通的编程语言。
同时，这也是为什么一开始的选型上，我并没有选择 Qt 而是尝试了彼时刚刚诞生的 Electron。

Electron 1.0 版本有着众多缺点，例如：发布的包体需要附加体积并不算小的依赖文件。
对我而言的优势是：我可以将丰富的 Web 开发经验，轻松地运用其中。
但很快就遇到了一些对于当时开发的项目的性能瓶颈（PS：也可能是我个人能力的不足），我开始寻觅更适合的框架。

兜兜转转又和 Qt 相遇了。
这可能也是一种“选择大于努力”了（笑。
C++ 固然难学，但是它自诞生以来活跃在各个领域经久不衰，这就足够坚定我认真学习它的想法了。

言归正传，现在就随我一起 从零开始 Qt 应用开发 吧！

安装

1. 下载安装程序

1.1 官网首页

打开官网 https://qt.io，点击顶部菜单上的帐户图标按钮。

1.2 登录

此时页面重定向到 https://login.qt.io/login，输入帐号密码登录。

Note

没有帐户的话，点击 Sign in 按钮下方的 Create Qt Account 链接跳转到注册页面先注册帐户。

1.3 下载

登录完成后，点击左侧边栏 Download，根据自己的平台点击对应在线安装程序（图片以 Windows x64 为例）。

2. 使用安装程序

2.1 打开安装程序

2.2 登录

点击 下一步，登录自己的 Qt 帐户。

2.3 开源义务

勾选 我已阅读并同意使用开源 Qt 的条款和条件 以及 我是个人用户，我不为任何公司使用 Qt 后，点击 下一步。

2.4 安装选项

指定安装 Qt 的目录（本文为：D:\Qt），取消勾选 用于 6.11 桌面开发的 Qt 库 并勾选下方的 自定义安装。

2.5 自定义安装

依次勾选 Qt → Qt 6.10.3 → MinGW 13.1.0 64-bit 和 Build Tools → MinGW 13.1.0 64-bit + CMake 3.30.5 + Ninja 1.12.1。

再勾选 Qt Creator → Qt Creator 19.0.0 + CDB Debugger Support + Debugging Tools for Windows，点击 下一步。

2.6 许可协议

阅读协议，勾选 I have read and agree to the terms contained in the license agreements. 后，点击 下一步。

2.7 开始菜单文件夹

填入开始菜单文件夹名称后，点击 下一步

2.8 安装总结

勾选 将常见文件类型关联到 Qt Creator，点击 安装。
