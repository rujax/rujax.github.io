到底该做个什么应用呢？

关于这个问题，我思考了许久。

原因主要有三：

  1. 我希望应用足够简单
  2. 尽可能多地展示 Qt 组件的使用
  3. 如果还稍微有一点点实用价值就更好了（笑

在反反复复提出又推翻无数想法后，无意间瞥到博客首页的 RSS 图标。
一个想法计上心头：不如，就做一个 RSS 阅读器，阅读我自己的博客文章，也算是一种自产自销了。
哈哈哈。。。

心动不如行动，那就开干呗！

创建项目

1. 打开 Qt Creator

点击右上角绿色按钮 创建项目… 后，依次选择 Application (Qt) → Qt Widgets Application（选择的原因后续呈上）后，点击 选择… 按钮。

2. Location

在 名称 处输入：RSSReader 后，选择项目路径，点击按钮 下一步。

3. 构建系统

默认选择是 CMake ，不用修改（古早时期，Qt 有一套自己的构建工具叫 QMake ，有兴趣的可以自行研究下）。

4. Details

默认填入的 Class name、Base class、Header file、Source file 均不用修改，唯独取消勾选 Generate form。

Important

这里解释一下 取消勾选 和 第1步选择 Qt Widgets Application 的原因： 目前，Qt 有多种实现 UI 的方式：

  1. Qt Quick + Qt Qml
  2. 在 Qt Widgets Application 中使用 .ui 文件
  3. 在 Qt Widgets Application 中手搓布局

我从刚开始接触 Web 前端，再到 iOS 应用开发，再到后面的桌面客户端应用开发，全部都是手搓布局，熟能生巧了。
我并非推荐手搓布局，也并非不推荐。
主要还是取决于哪一种方式更加顺手，效率更高。
建议大家多多尝试后再确定下来最适合自己的方式。

5. Translation

目前暂时不考虑多语言，语言 选择 <无> 后，点击 下一步 按钮。

6. 构建套件(Kit)

保持默认的勾选 Debug 即可，点击 下一步 按钮。

7. 汇总

版本控制系统，准备选择 Git ，但是打算在下一篇中介绍使用方法，因此这里选择 <无> 即可，点击 完成 按钮。

8. 创建成功

右键左上角 项目 下的 RSSReader，选择 展开全部，可以清晰地看到当前的项目文件结构：

对比一下此时 文件资源管理器 里项目文件结构：

可以看到 Qt Creator 中根据文件类型进行了分组，而在 文件资源管理器 中并没有分组，
多了一个 .qtcreator 文件夹存放配置和 build 文件夹存放构建项目后的结果。

正式编码前的准备

CMakeLists.txt

打开 CMakeLists.txt 文件，找到 project() 命令，在项目名称后添加版本信息：

# 修改之前
project(RSSReader LANGUAGES CXX)

# 修改之后
project(RSSReader VERSION 0.0.1 LANGUAGES CXX)

找到 qt_add_executable() 命令，剪切掉其中 .h 和 .cpp 的文件名，并在上方创建如图的内容：

# 修改之前
qt_add_executable(RSSReader
    WIN32 MACOSX_BUNDLE
    main.cpp
    mainwindow.cpp
    mainwindow.h
)

# 修改之后
set(SOURCES
    main.cpp
    mainwindow.cpp
)

set(HEADERS
    mainwindow.h
)

qt_add_executable(RSSReader
    WIN32 MACOSX_BUNDLE
    ${SOURCES}
    ${HEADERS}
)

这里的处理是为了方便后续添加更多 头文件 和 源文件 时一目了然，增加代码可读性。

进入 Qt 的编码世界

main.cpp

这是应用程序的入口文件，从初始化的代码中可以看到，它生成了一个 MainWindow 的实例，并调用 show() 实例方法显示了主窗口。

#include "mainwindow.h"

#include <QApplication>

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);
    MainWindow w;
    w.show();
    return QCoreApplication::exec();
}

这里，我们稍作调整，调用 resize 和 setMinimumSize 实例方法设置窗口最小尺寸为 800x600：

int main(int argc, char *argv[])
{
    ...

    MainWindow w;
    w.resize(800, 600);
    w.setMinimumSize(800, 600);
    w.show();

    ...
}

mainwindow.cpp

打开 mainwindow.cpp 文件，可以看到如下内容：

#include "mainwindow.h"

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
{}

MainWindow::~MainWindow() = default;

稍微调整一下 MainWindow::MainWindow() 构造函数的代码结构并使用 setWindowTitle() 方法设置窗口的标题：

...

MainWindow::MainWindow(QWidget *parent) : QMainWindow(parent)
{
    setWindowTitle("RSSReader");
}

...

初次启动应用

点击左下角绿色三角按钮，启动应用，查看当前代码修改后的效果。

可以看到一个尺寸为 800x600 且标题为 RSSReader 的空白窗口：

关闭窗口，接下来开始构建应用的布局。

mainwindow.h

打开 mainwindow.h 文件，可以看到如下的内容：

#ifndef MAINWINDOW_H
#define MAINWINDOW_H

#include <QMainWindow>

class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    explicit MainWindow(QWidget *parent = nullptr);
    ~MainWindow() override;
};
#endif // MAINWINDOW_H

稍微调整一下代码结构，然后在析构函数 ~MainWindow() override; 下方添加 private 类访问修饰符和 _buildUI() 的方法申明。

class MainWindow : public QMainWindow
{
    Q_OBJECT

public:
    explicit MainWindow(QWidget *parent = nullptr);
    ~MainWindow() override;

private:
    void _buildUI();
};

Tip

使用下划线 _ 开头命名私有变量、方法或其他私有成员是我个人的命名方法（忘了之前在哪篇文章看过的推荐了 (￣△￣；)

mainwindow.cpp

再次打开 mainwindow.cpp 文件。

MainWindow::MainWindow()

在构造函数 MainWindow::MainWindow() 中添加对 _buildUI() 方法的调用。

...

MainWindow::MainWindow(QWidget *parent) : QMainWindow(parent)
{
    setWindowTitle("RSSReader");

    _buildUI();
}

...

MainWindow::_buildUI()

然后在 MainWindow::~MainWindow() = default; 下面添加 _buildUI() 方法实现。

...

MainWindow::~MainWindow() = default;

void MainWindow::_buildUI()
{

}

在完成具体实现之前，我们首先需要了解一些关于 QBoxLayout 的知识。

简而言之，我们需要使用一个继承自 QBoxLayout 的 QVBoxLayout 作为主布局，
然后自上而下使用 addLayout 方法添加3个继承自 QBoxLayout 的 QHBoxLayout 分别作为顶部、中部、底部的布局。

#include "mainwindow.h"

// 文件顶部不要忘了引入 Qt 的库
#include <QBoxLayout>

...

void MainWindow::_buildUI()
{
    QHBoxLayout *topLayout = new QHBoxLayout;
    QHBoxLayout *middleLayout = new QHBoxLayout;
    QHBoxLayout *bottomLayout = new QHBoxLayout;

    QVBoxLayout *mainLayout = new QVBoxLayout;
    mainLayout->addLayout(topLayout, 2);
    mainLayout->addLayout(middleLayout, 9);
    mainLayout->addLayout(bottomLayout, 1);

    QWidget *container = new QWidget(this);
    container->setLayout(mainLayout);

    setCentralWidget(container);
}

上面的代码最后3行，创建了一个组件并且应用主布局，
最后通过 setCentralWidget 函数设置这个组件为主窗口的中心组件。

但是，此时启动应用的话，依然只能看到一个空白的窗口。
为了能够看到布局的效果，这里我们通过为每个布局使用 addWidget 方法添加一个 QLabel 用来显示内容。

#include "mainwindow.h"

#include <QBoxLayout>
#include <QLabel>

...

void MainWindow::_buildUI()
{
    QLabel *topLabel = new QLabel("top");
    topLabel->setAlignment(Qt::AlignCenter); // 设置文本居中
    topLabel->setStyleSheet("background-color: Aqua; font-size: 24px"); // 设置文本样式

    QHBoxLayout *topLayout = new QHBoxLayout;
    topLayout->addWidget(topLabel);

    QLabel *middleLabel = new QLabel("middle");
    middleLabel->setAlignment(Qt::AlignCenter);
    middleLabel->setStyleSheet("background-color: Violet; font-size: 24px");

    QHBoxLayout *middleLayout = new QHBoxLayout;
    middleLayout->addWidget(middleLabel);

    QLabel *bottomLabel = new QLabel("bottom");
    bottomLabel->setAlignment(Qt::AlignCenter);
    bottomLabel->setStyleSheet("background-color: SpringGreen; font-size: 24px");

    QHBoxLayout *bottomLayout = new QHBoxLayout;
    bottomLayout->addWidget(bottomLabel);

    QVBoxLayout *mainLayout = new QVBoxLayout;
    mainLayout->addLayout(topLayout, 2);
    mainLayout->addLayout(middleLayout, 9);
    mainLayout->addLayout(bottomLayout, 1);

    QWidget *container = new QWidget(this);
    container->setLayout(mainLayout);

    setCentralWidget(container);
}

为了更清楚地展示布局效果，标签使用了 CSS 。

启动应用，可以看到如图的窗口：

可以看出基础的布局已经呈现出来了，但是作为基础布局，布局之间的间距消除掉更好。

针对 mainLayout 调用 setSpacing 和 setContentsMargins 方法稍作修改：

...

void MainWindow::_buildUI()
{
    ...

    QVBoxLayout *mainLayout = new QVBoxLayout;
    mainLayout->setSpacing(0);
    mainLayout->setContentsMargins(0, 0, 0, 0);

    mainLayout->addLayout(topLayout, 2);
    mainLayout->addLayout(middleLayout, 9);
    mainLayout->addLayout(bottomLayout, 1);

    ...
}
