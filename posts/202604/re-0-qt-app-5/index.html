信号和槽

简介

Qt 使用一套名为 信号和槽 的技术来实现组件之间的通信。

简而言之：
当某个事件（例如：按钮点击）发生时就会发出该组件定义的信号，然后与这个信号建立连接的槽就会被调用。

优点

这个技术最直观的优点就是：松散耦合。

信号只负责发出信号，槽只负责接收信号，它们彼此之间并不需要知道对方的实现。
同时，一个信号可以与多个槽连接，一个槽也可以与多个信号连接，甚至信号与信号之间也可以连接。
这保证了 Qt 组件间通信的稳定、灵活、强大与易用。

使用

通过 QObject 这个所有 Qt 对象的基类的 connect 函数连接。

如上图所示：connect 函数共有 5 个重载版本。

Note

不同的连接方式有各自的 优缺点 ，根据自己的偏好使用即可。
本文 不使用基于字符串 的连接方式。

编程

本文将介绍 2 个版本的 connect 函数的使用。

订阅链接文本框

mainwindow.cpp

打开 mainwindow.cpp 文件，添加一段代码用来查看订阅链接文本框组件的值：

...

void MainWindow::_buildUI()
{
    ...

    _feedEdit->setPlaceholderText("请输入 RSS 链接");

    connect(_feedEdit, &QLineEdit::textChanged, [](const QString &text) {
        qDebug() << "text:" << text;
    });

    ...
}

通过将 QLineEdit 的 textChanged 信号和 lambda 连接，
每次订阅链接文本框组件的值发生变化时，都会在底部 应用程序输出 中打印出来。

订阅按钮

mainwindow.h

打开 mainwindow.h 文件，添加如下高亮代码：

...

class MainWindow : public QMainWindow
{
    ...

    void _buildUI();
    void _connectSlots();

private slots:
    void _subscribeButtonClicked();
};

简单描述下代码：

  1. 声明一个私有方法，将所有连接信号和槽的代码都放在里面（方便管理
  2. 声明一个私有槽，用来连接 QPushButton 的 clicked 信号（继承自 QAbstractButton ）

mainwindow.cpp

打开 mainwindow.cpp 文件，添加如下高亮代码：

...

#include <QScrollArea>
#include <QMessageBox>

#include "project.h"

MainWindow::MainWindow(QWidget *parent) : QMainWindow(parent)
{
    setWindowTitle("RSSReader");

    _buildUI();
    _connectSlots();
}

...

void MainWindow::_buildUI()
{
    ...
}

void MainWindow::_connectSlots()
{
    connect(_subscribeButton, &QPushButton::clicked, this, &MainWindow::_subscribeButtonClicked);
}

void MainWindow::_subscribeButtonClicked()
{
    QMessageBox::information(this, "单行编辑组件", _feedEdit->text());
}

简单描述下代码：

  1. include QMessageBox 组件类
  2. 在构造函数中调用 _connectSlots 方法
  3. 添加 _connectSlots 方法的实现，连接订阅按钮的点击信号和之前定义的私有槽
  4. 添加 _subscribeButtonClicked 私有槽的实现，调用 QMessageBox 的 information 函数，
    弹出一个对话框展示订阅链接文本框的值
